# <a name="compiler-error-cs0433"></a><span data-ttu-id="00221-103">Ошибка компилятора CS0433</span><span class="sxs-lookup"><span data-stu-id="00221-103">Compiler Error CS0433</span></span>

<span data-ttu-id="00221-104">Тип "имя типа 1" существует одновременно в "имя типа 2" и "имя типа 3"</span><span class="sxs-lookup"><span data-stu-id="00221-104">The type TypeName1 exists in both TypeName2 and TypeName3</span></span>  
  
 <span data-ttu-id="00221-105">Две разных сборки, на которые ссылается ваше приложение, содержат одинаковые пространство имен и тип, в результате чего возникает неоднозначность.</span><span class="sxs-lookup"><span data-stu-id="00221-105">Two different assemblies referenced in your application contain the same namespace and type, which produces ambiguity.</span></span>  
  
 <span data-ttu-id="00221-106">Чтобы устранить эту ошибку, используйте функцию псевдонима параметра компилятора [-reference (параметры компилятора C#)](../compiler-options/reference-compiler-option.md) или удалите ссылку на одну из этих сборок.</span><span class="sxs-lookup"><span data-stu-id="00221-106">To resolve this error, use the alias feature of the [-reference (C# Compiler Options)](../compiler-options/reference-compiler-option.md) compiler option or do not reference one of your assemblies.</span></span>  

<span data-ttu-id="00221-107">Эта ошибка также может возникнуть в веб-форм ASP.NET, если:</span><span class="sxs-lookup"><span data-stu-id="00221-107">This error can also occur in ASP.NET Web Forms if:</span></span>

* <span data-ttu-id="00221-108">Директива `@ Page` имеет атрибут `CodeFile` вместо атрибута `CodeBehind`.</span><span class="sxs-lookup"><span data-stu-id="00221-108">The `@ Page` directive has a `CodeFile` attribute when it should be a `CodeBehind` attribute.</span></span>

 <span data-ttu-id="00221-110">Эта ошибка также может возникать, если код находится в папке APP_CODE, которая не должна быть в папке APP_CODE.</span><span class="sxs-lookup"><span data-stu-id="00221-110">This error can also occur if code is in an APP_CODE folder that should not be in an APP_CODE folder.</span></span>

## <a name="example"></a><span data-ttu-id="00221-115">Пример</span><span class="sxs-lookup"><span data-stu-id="00221-115">Example</span></span>  

 <span data-ttu-id="00221-116">В следующем примере возникает ошибка CS0433.</span><span class="sxs-lookup"><span data-stu-id="00221-116">The following example generates CS0433.</span></span>  
  
```csharp  
// CS0433_3.cs  
// compile with: /reference:cs0433_1.dll /reference:cs0433_2.dll  
using TypeBindConflicts;  
public class Test
{  
   public static void Main()
   {  
      AggPubImpAggPubImp n6 = new AggPubImpAggPubImp();   // CS0433  
   }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="00221-117">Пример</span><span class="sxs-lookup"><span data-stu-id="00221-117">Example</span></span>  

 <span data-ttu-id="00221-118">В следующем примере показано, как можно использовать функцию псевдонима параметра компилятора **/reference** для устранения возникшей ошибки CS0433.</span><span class="sxs-lookup"><span data-stu-id="00221-118">The following example shows how you can use the alias feature of the **/reference** compiler option to resolve this CS0433 error.</span></span>

```csharp  
// compile with: /reference:cs0433_1.dll /reference:TypeBindConflicts=cs0433_2.dll

{  
   public static void Main()
   {  
      AggPubImpAggPubImp n6 = new AggPubImpAggPubImp();  
   }  
}  
```
