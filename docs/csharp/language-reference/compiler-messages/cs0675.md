# <a name="compiler-warning-level-3-cs0675"></a><span data-ttu-id="3adf8-103">Предупреждение компилятора (уровень 3) CS0675</span><span class="sxs-lookup"><span data-stu-id="3adf8-103">Compiler Warning (level 3) CS0675</span></span>

<span data-ttu-id="3adf8-104">Побитовый оператор "ИЛИ" применен к операнду, расширенному знаком; рекомендуется предварительное приведение к меньшему беззнаковому типу</span><span class="sxs-lookup"><span data-stu-id="3adf8-104">Bitwise-or operator used on a sign-extended operand; consider casting to a smaller unsigned type first</span></span>  
  
 <span data-ttu-id="3adf8-105">Компилятор неявно расширил знаком переменную, после чего использовал полученное значение в побитовой операции ИЛИ.</span><span class="sxs-lookup"><span data-stu-id="3adf8-105">The compiler implicitly widened and sign-extended a variable, and then used the resulting value in a bitwise OR operation.</span></span> <span data-ttu-id="3adf8-106">Это может привести к непредвиденному поведению.</span><span class="sxs-lookup"><span data-stu-id="3adf8-106">This can result in unexpected behavior.</span></span>  
  
 <span data-ttu-id="3adf8-107">В следующем примере возникает предупреждение CS0675:</span><span class="sxs-lookup"><span data-stu-id="3adf8-107">The following sample generates CS0675:</span></span>  
  
```csharp  
// CS0675.cs  
// compile with: /W:3  
using System;  
  
public class sign  
{  
   public static void Main()  
   {  
      int hi = 1;  
      int lo = 1;  
      long value = (((long)hi) << 32) | lo;              // CS0675  
      // try the following line instead  
      // long value = (((long)hi) << 32) | ((uint)lo);   // correct  
   }  
}  
```
