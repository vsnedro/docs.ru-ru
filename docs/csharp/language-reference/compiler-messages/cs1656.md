# <a name="compiler-error-cs1656"></a><span data-ttu-id="ea154-103">Ошибка компилятора CS1656</span><span class="sxs-lookup"><span data-stu-id="ea154-103">Compiler Error CS1656</span></span>

<span data-ttu-id="ea154-104">Невозможно присвоить значение переменной "переменная", так как она имеет тип "тип переменной только для чтения"</span><span class="sxs-lookup"><span data-stu-id="ea154-104">Cannot assign to 'variable' because it is a 'read-only variable type'</span></span>  
  
 <span data-ttu-id="ea154-105">Эта ошибка возникает в случае присвоения значения переменной в контексте только для чтения.</span><span class="sxs-lookup"><span data-stu-id="ea154-105">This error occurs when an assignment to variable occurs in a read-only context.</span></span> <span data-ttu-id="ea154-106">К контекстам только для чтения относятся переменные итерации [foreach](../keywords/foreach-in.md), переменные [using](../keywords/using-statement.md) и переменные [fixed](../keywords/fixed-statement.md).</span><span class="sxs-lookup"><span data-stu-id="ea154-106">Read-only contexts include [foreach](../keywords/foreach-in.md) iteration variables, [using](../keywords/using-statement.md) variables, and [fixed](../keywords/fixed-statement.md) variables.</span></span> <span data-ttu-id="ea154-107">Чтобы устранить эту ошибку, не присваивайте значения переменной оператора в блоках `using`, операторах `foreach` и операторах `fixed`.</span><span class="sxs-lookup"><span data-stu-id="ea154-107">To resolve this error, avoid assignments to a statement variable in `using` blocks, `foreach` statements, and `fixed` statements.</span></span>  
  
## <a name="example"></a><span data-ttu-id="ea154-108">Пример</span><span class="sxs-lookup"><span data-stu-id="ea154-108">Example</span></span>  

 <span data-ttu-id="ea154-109">В следующем примере возникает ошибка CS1656 из-за попытки полностью заменить элементы коллекции в цикле `foreach`.</span><span class="sxs-lookup"><span data-stu-id="ea154-109">The following example generates error CS1656 because it tries to replace complete elements of a collection inside a `foreach` loop.</span></span> <span data-ttu-id="ea154-110">Чтобы обойти эту ошибку, можно попробовать заменить цикл `foreach` на цикл [for](../keywords/for.md).</span><span class="sxs-lookup"><span data-stu-id="ea154-110">One way to work around the error is to change the `foreach` loop to a [for](../keywords/for.md) loop.</span></span> <span data-ttu-id="ea154-111">Кроме того, можно изменить члены существующего элемента. Этот способ не показан здесь и подходит только для классов, но не для структур.</span><span class="sxs-lookup"><span data-stu-id="ea154-111">Another way, not shown here, is to modify the members of the existing element; this is possible with classes, but not with structs.</span></span>  
  
```csharp  
using System;  
using System.Collections;  
using System.Collections.Generic;  
using System.Text;  
  
namespace CS1656_2  
{  
  
    class Book  
    {  
        public string Title;  
        public string Author;  
        public double Price;  
        public Book(string t, string a, double p)  
        {  
            Title=t;  
            Author=a;  
            Price=p;  
  
        }  
    }  
  
    class Program  
    {  
        private List<Book> list;  
        static void Main(string[] args)  
        {  
            Program prog = new Program();  
            prog.list = new List<Book>();  
            prog.list.Add(new Book ("The C# Programming Language",  
                                    "Hejlsberg, Wiltamuth, Golde",  
                                     29.95));  
            prog.list.Add(new Book ("The C++ Programming Language",  
                                    "Stroustrup",  
                                     29.95));  
            prog.list.Add(new Book ("The C Programming Language",  
                                    "Kernighan, Ritchie",  
                                    29.95));  
            foreach(Book b in prog.list)  
            {  
                // Cannot modify an entire element in a foreach loop
                // even with reference types.  
                // Use a for or while loop instead  
                if (b.Title == "The C Programming Language")  
                    // Cannot assign to 'b' because it is a 'foreach
                    // iteration variable'  
                    b = new Book("Programming Windows, 5th Ed.", "Petzold", 29.95); //CS1656  
            }  
  
            //With a for loop you can modify elements  
            //for(int x = 0; x < prog.list.Count; x++)  
            //{  
            //    if(prog.list[x].Title== "The C Programming Language")  
            //        prog.list[x] = new Book("Programming Windows, 5th Ed.", "Petzold", 29.95);  
            //}  
            //foreach(Book b in prog.list)  
            //    Console.WriteLine(b.Title);  
  
        }  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="ea154-112">Пример</span><span class="sxs-lookup"><span data-stu-id="ea154-112">Example</span></span>  

 <span data-ttu-id="ea154-113">В следующем примере показано, как ошибка CS1656 может возникать в других контекстах, помимо цикла `foreach`:</span><span class="sxs-lookup"><span data-stu-id="ea154-113">The following sample demonstrates how CS1656 can be generated in other contexts besides a `foreach` loop:</span></span>  
  
```csharp  
// CS1656.cs  
// compile with: /unsafe  
using System;  
  
class C : IDisposable  
{  
    public void Dispose() { }  
}  
  
class CMain  
{  
    unsafe public static void Main()  
    {  
        using (C c = new C())  
        {  
            // Cannot assign to 'c' because it is a 'using variable'  
            c = new C(); // CS1656  
        }  
  
        int[] ary = new int[] { 1, 2, 3, 4 };  
        fixed (int* p = ary)  
        {  
            // Cannot assign to 'p' because it is a 'fixed variable'  
            p = null; // CS1656  
        }  
    }  
}  
```
