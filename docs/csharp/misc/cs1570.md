<span data-ttu-id="e1f80-104">XML-комментарий для "конструкция" содержит некорректный XML — "причина"</span><span class="sxs-lookup"><span data-stu-id="e1f80-104">XML comment on 'construct' has badly formed XML — 'reason'</span></span>  
  
 <span data-ttu-id="e1f80-105">При использовании [-doc](../language-reference/compiler-options/doc-compiler-option.md)любые комментарии в исходном коде должны быть в формате XML.</span><span class="sxs-lookup"><span data-stu-id="e1f80-105">When using [-doc](../language-reference/compiler-options/doc-compiler-option.md), any comments in the source code must be in XML.</span></span> <span data-ttu-id="e1f80-106">Любая ошибка разметки XML вызывает предупреждение CS1570.</span><span class="sxs-lookup"><span data-stu-id="e1f80-106">Any error with your XML markup will generate CS1570.</span></span> <span data-ttu-id="e1f80-107">Пример:</span><span class="sxs-lookup"><span data-stu-id="e1f80-107">For example:</span></span>  
  
- <span data-ttu-id="e1f80-108">Если вы передаете строку в **cref**, например в [\<exception>](../programming-guide/xmldoc/exception.md) тег, строка должна быть заключена в двойные кавычки.</span><span class="sxs-lookup"><span data-stu-id="e1f80-108">If you are passing a string to a **cref**, such as in an [\<exception>](../programming-guide/xmldoc/exception.md) tag, the string must be enclosed in double quotation marks.</span></span>  
  
- <span data-ttu-id="e1f80-109">Если вы используете тег, например [\<seealso>](../programming-guide/xmldoc/seealso.md) , который не имеет закрывающего тега, перед закрывающей угловой скобкой необходимо указать прямую косую черту.</span><span class="sxs-lookup"><span data-stu-id="e1f80-109">If you are using a tag, such as [\<seealso>](../programming-guide/xmldoc/seealso.md), which does not have a closing tag, you must specify a forward slash before the closing angle bracket.</span></span>  
  
- <span data-ttu-id="e1f80-110">Если в тексте описания необходимо использовать символ «больше или меньше», необходимо представить их с помощью **&gt;** или **&lt;** .</span><span class="sxs-lookup"><span data-stu-id="e1f80-110">If you need to use a greater-than or less-than symbol in the text of description, you need to represent them with **&gt;** or **&lt;**.</span></span>  
  
- <span data-ttu-id="e1f80-111">Атрибут файла или пути в [\<include>](../programming-guide/xmldoc/include.md) теге отсутствует или имеет неправильный формат.</span><span class="sxs-lookup"><span data-stu-id="e1f80-111">The file or path attribute on an [\<include>](../programming-guide/xmldoc/include.md) tag was missing or improperly formed.</span></span>
