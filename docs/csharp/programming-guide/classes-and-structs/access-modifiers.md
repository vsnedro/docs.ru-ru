- <span data-ttu-id="b9ea4-108">[public](../../language-reference/keywords/public.md): Доступ к типу или члену возможен из любого другого кода в той же сборке или другой сборке, ссылающейся на него.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-108">[public](../../language-reference/keywords/public.md): The type or member can be accessed by any other code in the same assembly or another assembly that references it.</span></span>

- <span data-ttu-id="b9ea4-111">[internal](../../language-reference/keywords/internal.md): Доступ к типу или члену возможен из любого кода в той же сборке, но не из другой сборки.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-111">[internal](../../language-reference/keywords/internal.md): The type or member can be accessed by any code in the same assembly, but not from another assembly.</span></span>

- <span data-ttu-id="b9ea4-113">[private protected](../../language-reference/keywords/private-protected.md): доступ к типу или члену возможен только из его объявляющей сборки из кода в том же `class` либо в типе, производном от этого `class`.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-113">[private protected](../../language-reference/keywords/private-protected.md): The type or member can be accessed only within its declaring assembly, by code in the same `class` or in a type that is derived from that `class`.</span></span>

## <a name="class-and-struct-accessibility"></a><span data-ttu-id="b9ea4-117">Доступность классов и структур</span><span class="sxs-lookup"><span data-stu-id="b9ea4-117">Class and struct accessibility</span></span>  

<span data-ttu-id="b9ea4-118">Классы и структуры, объявленные непосредственно в пространстве имен (другими словами, не вложенные в другие классы или структуры), могут быть `public` или `internal`.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-118">Classes and structs declared directly within a namespace (in other words, that aren't nested within other classes or structs) can be either `public` or `internal`.</span></span> <span data-ttu-id="b9ea4-119">Если модификатор доступа не указан, по умолчанию используется `Internal`.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-119">`Internal` is the default if no access modifier is specified.</span></span>  

<span data-ttu-id="b9ea4-120">Члены структуры, включая вложенные классы и структуры, могут быть объявлены как `public`, `internal` или `private`.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-120">Struct members, including nested classes and structs, can be declared `public`, `internal`, or `private`.</span></span> <span data-ttu-id="b9ea4-121">Члены класса, включая вложенные классы и структуры, могут быть `public`, `protected internal`, `protected`, `internal`, `private protected` или `private`.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-121">Class members, including nested classes and structs, can be `public`, `protected internal`, `protected`, `internal`, `private protected`, or `private`.</span></span> <span data-ttu-id="b9ea4-122">Члены класса и структуры, включая вложенные классы и структуры, по умолчанию имеют доступ `private`.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-122">Class and struct members,  including nested classes and structs, have `private` access by default.</span></span> <span data-ttu-id="b9ea4-123">Закрытые вложенные типы недоступны за пределами типа, в котором содержатся.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-123">Private nested types aren't accessible from outside the containing type.</span></span>

<span data-ttu-id="b9ea4-124">Производные классы не могут быть более доступны, чем соответствующие базовые типы.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-124">Derived classes can't have greater accessibility than their base types.</span></span> <span data-ttu-id="b9ea4-125">Невозможно объявить открытый класс `B`, производный от внутреннего класса `A`.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-125">You can't declare a public class `B` that derives from an internal class `A`.</span></span> <span data-ttu-id="b9ea4-126">Если бы это было возможно, класс `A` стал бы открытым, так как все члены `protected` или `internal` класса `A` были бы доступны из производного класса.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-126">If allowed, it would have the effect of making `A` public, because all `protected` or `internal` members of `A` are accessible from the derived class.</span></span>

## <a name="class-and-struct-member-accessibility"></a><span data-ttu-id="b9ea4-129">Доступность членов классов и структур</span><span class="sxs-lookup"><span data-stu-id="b9ea4-129">Class and struct member accessibility</span></span>  

<span data-ttu-id="b9ea4-130">Члены класса (включая вложенные классы и структуры) можно объявлять с любым из шести типов доступа.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-130">Class members (including nested classes and structs) can be declared with any of the six types of access.</span></span> <span data-ttu-id="b9ea4-131">Члены структуры не могут быть объявлены как `protected`, `protected internal` или `private protected`, так как структуры не поддерживают наследование.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-131">Struct members can't be declared as `protected`, `protected internal`, or `private protected` because structs don't support inheritance.</span></span>

<span data-ttu-id="b9ea4-132">Как правило, уровень доступности члена не может быть выше уровня доступности типа, в который он входит.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-132">Normally, the accessibility of a member isn't greater than the accessibility of the type that contains it.</span></span> <span data-ttu-id="b9ea4-133">При этом член `public` внутреннего класса может быть доступен за пределами сборки, если он реализует методы интерфейса или переопределяет виртуальные методы, определенные в открытом базовом классе.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-133">However, a `public` member of an internal class might be accessible from outside the assembly if the member implements interface methods or overrides virtual methods that are defined in a public base class.</span></span>

<span data-ttu-id="b9ea4-134">Тип любого поля, свойства или события члена должен иметь по меньшей мере такой же уровень доступности, как у самого члена.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-134">The type of any member field, property, or event must be at least as accessible as the member itself.</span></span> <span data-ttu-id="b9ea4-135">Точно так же тип возвращаемого значения и типы параметров любого метода, индексатора или делегата должен иметь по меньшей мере такой же уровень доступности, как у самого члена.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-135">Similarly, the return type and the parameter types of any method, indexer, or delegate must be at least as accessible as the member itself.</span></span> <span data-ttu-id="b9ea4-136">Например, невозможно иметь метод `public` `M`, возвращающий класс `C`, если только `C` также не является `public`.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-136">For example, you can't have a `public` method `M` that returns a class `C` unless `C` is also `public`.</span></span> <span data-ttu-id="b9ea4-137">Аналогичным образом, невозможно иметь свойство `protected` типа `A`, если `A` объявлен как `private`.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-137">Likewise, you can't have a `protected` property of type `A` if `A` is declared as `private`.</span></span>

<span data-ttu-id="b9ea4-138">Пользовательские операторы всегда должны объявляться как `public` и `static`.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-138">User-defined operators must always be declared as `public` and `static`.</span></span> <span data-ttu-id="b9ea4-139">Для получения дополнительной информации см. раздел [Перегрузка операторов](../../language-reference/operators/operator-overloading.md).</span><span class="sxs-lookup"><span data-stu-id="b9ea4-139">For more information, see [Operator overloading](../../language-reference/operators/operator-overloading.md).</span></span>

<span data-ttu-id="b9ea4-143">Интерфейсы, объявляемые непосредственно в пространстве имен, могут быть `public` или `internal`. Как и в случае с классами и структурами, для интерфейсов по умолчанию задается доступ `internal`.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-143">Interfaces declared directly within a namespace can be `public` or `internal` and, just like classes and structs, interfaces default to `internal` access.</span></span> <span data-ttu-id="b9ea4-144">Члены интерфейса по умолчанию являются `public`, так как интерфейс как раз и создан для того, чтобы обеспечить другим типам доступ к классу или структуре.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-144">Interface members are `public` by default because the purpose of an interface is to enable other types to access a class or struct.</span></span> <span data-ttu-id="b9ea4-145">Объявления членов интерфейса могут включать любой модификатор доступа.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-145">Interface member declarations may include any access modifier.</span></span> <span data-ttu-id="b9ea4-146">Это удобнее всего использовать для статических методов, чтобы предоставить общие реализации, необходимые всем объектам, реализующим класс.</span><span class="sxs-lookup"><span data-stu-id="b9ea4-146">This is most useful for static methods to provide common implementations needed by all implementors of a class.</span></span>
