# <a name="enabling-a-data-source-for-linq-querying"></a><span data-ttu-id="981c6-103">Включение источника данных для запросов LINQ</span><span class="sxs-lookup"><span data-stu-id="981c6-103">Enabling a Data Source for LINQ Querying</span></span>

<span data-ttu-id="981c6-104">Существуют различные способы расширения LINQ для поддержки запросов к любому источнику данных в шаблоне LINQ.</span><span class="sxs-lookup"><span data-stu-id="981c6-104">There are various ways to extend LINQ to enable any data source to be queried in the LINQ pattern.</span></span> <span data-ttu-id="981c6-105">Источник данных может быть, например, структурой данных, веб-службой, файловой системой или базой данных.</span><span class="sxs-lookup"><span data-stu-id="981c6-105">The data source might be a data structure, a Web service, a file system, or a database, to name some.</span></span> <span data-ttu-id="981c6-106">Шаблон LINQ позволяет клиентам легко выполнять запросы к источнику данных, для которого включен LINQ-запрос, поскольку синтаксис и шаблон запроса не меняются.</span><span class="sxs-lookup"><span data-stu-id="981c6-106">The LINQ pattern makes it easy for clients to query a data source for which LINQ querying is enabled, because the syntax and pattern of the query does not change.</span></span> <span data-ttu-id="981c6-107">Далее перечислены способы расширения LINQ для этих источников данных.</span><span class="sxs-lookup"><span data-stu-id="981c6-107">The ways in which LINQ can be extended to these data sources include the following:</span></span>  
  
- <span data-ttu-id="981c6-108">Реализация интерфейса <xref:System.Collections.Generic.IEnumerable%601> в типе для выполнения запросов LINQ to Objects этого типа.</span><span class="sxs-lookup"><span data-stu-id="981c6-108">Implementing the <xref:System.Collections.Generic.IEnumerable%601> interface in a type to enable LINQ to Objects querying of that type.</span></span>  
  
- <span data-ttu-id="981c6-109">Создание методов стандартных операторов запроса, таких как <xref:System.Linq.Enumerable.Where%2A> и <xref:System.Linq.Enumerable.Select%2A>, расширяющих тип, для поддержки настраиваемых запросов LINQ этого типа.</span><span class="sxs-lookup"><span data-stu-id="981c6-109">Creating standard query operator methods such as <xref:System.Linq.Enumerable.Where%2A> and <xref:System.Linq.Enumerable.Select%2A> that extend a type, to enable custom LINQ querying of that type.</span></span>  
  
- <span data-ttu-id="981c6-110">Создание поставщика для источника данных, который реализует интерфейс <xref:System.Linq.IQueryable%601>.</span><span class="sxs-lookup"><span data-stu-id="981c6-110">Creating a provider for your data source that implements the <xref:System.Linq.IQueryable%601> interface.</span></span> <span data-ttu-id="981c6-111">Поставщик, реализующий этот интерфейс, получает запросы LINQ в виде деревьев выражения, которые он может выполнять любым способом, например удаленно.</span><span class="sxs-lookup"><span data-stu-id="981c6-111">A provider that implements this interface receives LINQ queries in the form of expression trees, which it can execute in a custom way, for example remotely.</span></span>  
  
- <span data-ttu-id="981c6-112">Создание поставщика для источника данных, который использует преимущества существующей технологии LINQ.</span><span class="sxs-lookup"><span data-stu-id="981c6-112">Creating a provider for your data source that takes advantage of an existing LINQ technology.</span></span> <span data-ttu-id="981c6-113">Такой поставщик позволит выполнять не только запросы, но и операции вставки, обновления, удаления и сопоставления для определяемых пользователем типов.</span><span class="sxs-lookup"><span data-stu-id="981c6-113">Such a provider would enable not only querying, but also insert, update, and delete operations and mapping for user-defined types.</span></span>  
  
 <span data-ttu-id="981c6-114">Эти варианты рассматриваются далее.</span><span class="sxs-lookup"><span data-stu-id="981c6-114">This topic discusses these options.</span></span>  
  
## <a name="how-to-enable-linq-querying-of-your-data-source"></a><span data-ttu-id="981c6-115">Включение поддержки запросов LINQ к пользовательскому источнику данных</span><span class="sxs-lookup"><span data-stu-id="981c6-115">How to Enable LINQ Querying of Your Data Source</span></span>  
  
### <a name="in-memory-data"></a><span data-ttu-id="981c6-116">Данные в памяти</span><span class="sxs-lookup"><span data-stu-id="981c6-116">In-Memory Data</span></span>  

 <span data-ttu-id="981c6-117">Существует два способа включения поддержки запросов LINQ к данным, находящимся в памяти.</span><span class="sxs-lookup"><span data-stu-id="981c6-117">There are two ways you can enable LINQ querying of in-memory data.</span></span> <span data-ttu-id="981c6-118">Если данные имеют тип, который реализует <xref:System.Collections.Generic.IEnumerable%601>, можно выполнить запрос данных с помощью LINQ to Objects.</span><span class="sxs-lookup"><span data-stu-id="981c6-118">If the data is of a type that implements <xref:System.Collections.Generic.IEnumerable%601>, you can query the data by using LINQ to Objects.</span></span> <span data-ttu-id="981c6-119">Если добавление перечисления к типу с помощью реализации интерфейса <xref:System.Collections.Generic.IEnumerable%601> не имеет смысла, можно определить методы стандартных операторов запроса LINQ в типе или создать методы стандартных операторов запроса LINQ, которые расширяют тип.</span><span class="sxs-lookup"><span data-stu-id="981c6-119">If it does not make sense to enable enumeration of your type by implementing the <xref:System.Collections.Generic.IEnumerable%601> interface, you can define LINQ standard query operator methods in that type or create LINQ standard query operator methods that extend the type.</span></span> <span data-ttu-id="981c6-120">Пользовательские реализации стандартных операторов запроса должны использовать отложенное выполнение для возврата результатов.</span><span class="sxs-lookup"><span data-stu-id="981c6-120">Custom implementations of the standard query operators should use deferred execution to return the results.</span></span>  
  
### <a name="remote-data"></a><span data-ttu-id="981c6-121">Удаленные данные</span><span class="sxs-lookup"><span data-stu-id="981c6-121">Remote Data</span></span>  

 <span data-ttu-id="981c6-122">Лучшим вариантом для поддержки запросов LINQ к удаленному источнику данных является реализация интерфейса <xref:System.Linq.IQueryable%601>.</span><span class="sxs-lookup"><span data-stu-id="981c6-122">The best option for enabling LINQ querying of a remote data source is to implement the <xref:System.Linq.IQueryable%601> interface.</span></span> <span data-ttu-id="981c6-123">Однако это отличается от расширения поставщика, например [!INCLUDE[vbtecdlinq](~/includes/vbtecdlinq-md.md)] для источника данных.</span><span class="sxs-lookup"><span data-stu-id="981c6-123">However, this differs from extending a provider such as [!INCLUDE[vbtecdlinq](~/includes/vbtecdlinq-md.md)] for a data source.</span></span> <span data-ttu-id="981c6-124">В Visual Studio 2008 отсутствуют модели поставщика для расширения существующих технологий LINQ, таких как [!INCLUDE[vbtecdlinq](~/includes/vbtecdlinq-md.md)], до других типов источников данных.</span><span class="sxs-lookup"><span data-stu-id="981c6-124">No provider models for extending existing LINQ technologies, such as [!INCLUDE[vbtecdlinq](~/includes/vbtecdlinq-md.md)], to other types of data source are available in Visual Studio 2008.</span></span>
  
## <a name="iqueryable-linq-providers"></a><span data-ttu-id="981c6-125">Поставщики IQueryable LINQ</span><span class="sxs-lookup"><span data-stu-id="981c6-125">IQueryable LINQ Providers</span></span>  

 <span data-ttu-id="981c6-126">Поставщики LINQ, реализующие <xref:System.Linq.IQueryable%601>, могут существенно различаться по сложности.</span><span class="sxs-lookup"><span data-stu-id="981c6-126">LINQ providers that implement <xref:System.Linq.IQueryable%601> can vary widely in their complexity.</span></span> <span data-ttu-id="981c6-127">В этом разделе рассматриваются различные уровни сложности.</span><span class="sxs-lookup"><span data-stu-id="981c6-127">This section discusses the different levels of complexity.</span></span>  
  
 <span data-ttu-id="981c6-128">Менее сложные поставщики `IQueryable` могут взаимодействовать с одним методом веб-службы.</span><span class="sxs-lookup"><span data-stu-id="981c6-128">A less complex `IQueryable` provider might interface with a single method of a Web service.</span></span> <span data-ttu-id="981c6-129">Этот тип поставщиков является очень специализированным, так как он требует наличия определенных сведений в обрабатываемых запросах.</span><span class="sxs-lookup"><span data-stu-id="981c6-129">This type of provider is very specific because it expects specific information in the queries that it handles.</span></span> <span data-ttu-id="981c6-130">Он имеет закрытую систему типов, возможно, предоставляя один тип результата.</span><span class="sxs-lookup"><span data-stu-id="981c6-130">It has a closed type system, perhaps exposing a single result type.</span></span> <span data-ttu-id="981c6-131">Большая часть выполнения запроса происходит локально, например с помощью реализации <xref:System.Linq.Enumerable> стандартных операторов запроса.</span><span class="sxs-lookup"><span data-stu-id="981c6-131">Most of the execution of the query occurs locally, for example by using the <xref:System.Linq.Enumerable> implementations of the standard query operators.</span></span> <span data-ttu-id="981c6-132">Менее сложный поставщик может проверять только одно выражение вызова метода в дереве выражений, которое представляет запрос, и оставляет оставшуюся логику запроса обрабатываться в другом месте.</span><span class="sxs-lookup"><span data-stu-id="981c6-132">A less complex provider might examine only one method call expression in the expression tree that represents the query, and let the remaining logic of the query be handled elsewhere.</span></span>  
  
 <span data-ttu-id="981c6-133">Поставщик `IQueryable` средней сложности может использовать источник данных, который имеет ограниченно выразительный язык запросов.</span><span class="sxs-lookup"><span data-stu-id="981c6-133">An `IQueryable` provider of medium complexity might target a data source that has a partially expressive query language.</span></span> <span data-ttu-id="981c6-134">Если он предназначен для веб-службы, он может взаимодействовать с несколькими методами веб-службы и выбирать метод для вызова на основании вопроса, содержащегося в запросе.</span><span class="sxs-lookup"><span data-stu-id="981c6-134">If it targets a Web service, it might interface with more than one method of the Web service and select the method to call based on the question that the query poses.</span></span> <span data-ttu-id="981c6-135">Поставщик средней сложности будет иметь более богатую систему типов, чем простой поставщик, но она по-прежнему будет фиксированной.</span><span class="sxs-lookup"><span data-stu-id="981c6-135">A provider of medium complexity would have a richer type system than a simple provider, but it would still be a fixed type system.</span></span> <span data-ttu-id="981c6-136">Например, поставщик может предоставлять типы, имеющие связи "один ко многим", которые можно обойти, но он не предоставляет технологии отображения для определяемых пользователем типов.</span><span class="sxs-lookup"><span data-stu-id="981c6-136">For example, the provider might expose types that have one-to-many relationships that can be traversed, but it would not provide mapping technology for user-defined types.</span></span>  
  
 <span data-ttu-id="981c6-137">Сложный поставщик `IQueryable`, например поставщик [!INCLUDE[vbtecdlinq](~/includes/vbtecdlinq-md.md)], может переводить завершенные запросы LINQ на выразительный язык запросов, например SQL.</span><span class="sxs-lookup"><span data-stu-id="981c6-137">A complex `IQueryable` provider, such as the [!INCLUDE[vbtecdlinq](~/includes/vbtecdlinq-md.md)] provider, might translate complete LINQ queries to an expressive query language, such as SQL.</span></span> <span data-ttu-id="981c6-138">Сложные поставщики более общие, чем менее сложные, поскольку они могут обрабатывать более широкий диапазон вопросов в запросе.</span><span class="sxs-lookup"><span data-stu-id="981c6-138">A complex provider is more general than a less complex provider, because it can handle a wider variety of questions in the query.</span></span> <span data-ttu-id="981c6-139">Он также имеет открытую систему типов и поэтому должен содержать расширяемую инфраструктуру для сопоставления определяемых пользователем типов.</span><span class="sxs-lookup"><span data-stu-id="981c6-139">It also has an open type system and therefore must contain extensive infrastructure to map user-defined types.</span></span> <span data-ttu-id="981c6-140">Разработка сложного поставщика требует значительных усилий.</span><span class="sxs-lookup"><span data-stu-id="981c6-140">Developing a complex provider requires a significant amount of effort.</span></span>
