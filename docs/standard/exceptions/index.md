<span data-ttu-id="9aae6-105">Необходимо реализовать возможность единообразной обработки приложениями ошибок, происходящих во время выполнения.</span><span class="sxs-lookup"><span data-stu-id="9aae6-105">Applications must be able to handle errors that occur during execution in a consistent manner.</span></span> <span data-ttu-id="9aae6-106">Среда .NET предоставляет модель для единообразного уведомления приложений об ошибках: операции .NET информируют о сбое посредством выдачи исключений.</span><span class="sxs-lookup"><span data-stu-id="9aae6-106">.NET provides a model for notifying applications of errors in a uniform way: .NET operations indicate failure by throwing exceptions.</span></span>

## <a name="exceptions"></a><span data-ttu-id="9aae6-107">Исключения</span><span class="sxs-lookup"><span data-stu-id="9aae6-107">Exceptions</span></span>

<span data-ttu-id="9aae6-108">Исключение — это любое состояние ошибки или непредвиденное поведение, возникающее при выполнении программы.</span><span class="sxs-lookup"><span data-stu-id="9aae6-108">An exception is any error condition or unexpected behavior that is encountered by an executing program.</span></span> <span data-ttu-id="9aae6-109">Исключения могут возникать из-за сбоя в вашем или вызываемом коде (например, в общей библиотеке), недоступности ресурсов ОС, неожиданных состояний, возникающих в среде выполнения (например, код, который невозможно проверить) и по другим причинам.</span><span class="sxs-lookup"><span data-stu-id="9aae6-109">Exceptions can be thrown because of a fault in your code or in code that you call (such as a shared library), unavailable operating system resources, unexpected conditions that the runtime encounters (such as code that can't be verified), and so on.</span></span> <span data-ttu-id="9aae6-110">После некоторых из этих состояний приложение может восстановиться, после других — нет.</span><span class="sxs-lookup"><span data-stu-id="9aae6-110">Your application can recover from some of these conditions, but not from others.</span></span> <span data-ttu-id="9aae6-111">В большинстве случаев вы можете выполнить восстановление после большинства исключений в приложении, но не после исключений среды выполнения.</span><span class="sxs-lookup"><span data-stu-id="9aae6-111">Although you can recover from most application exceptions, you can't recover from most runtime exceptions.</span></span>

<span data-ttu-id="9aae6-112">В .NET исключение — это объект, наследуемый от класса <xref:System.Exception?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="9aae6-112">In .NET, an exception is an object that inherits from the <xref:System.Exception?displayProperty=nameWithType> class.</span></span> <span data-ttu-id="9aae6-113">Исключение создается из области кода, где произошла проблема.</span><span class="sxs-lookup"><span data-stu-id="9aae6-113">An exception is thrown from an area of code where a problem has occurred.</span></span> <span data-ttu-id="9aae6-114">Исключение передается вверх по стеку до тех пор, пока его не обработает приложение либо программа не завершится.</span><span class="sxs-lookup"><span data-stu-id="9aae6-114">The exception is passed up the stack until the application handles it or the program terminates.</span></span>

<span data-ttu-id="9aae6-116">Традиционно модели обработки ошибок разных языков программирования основываются либо на уникальном для языка способе обнаружения ошибок и отыскании для них обработчиков, либо на механизме обработки ошибок, предоставляемом операционной системой.</span><span class="sxs-lookup"><span data-stu-id="9aae6-116">Traditionally, a language's error-handling model relied on either the language's unique way of detecting errors and locating handlers for them, or on the error-handling mechanism provided by the operating system.</span></span> <span data-ttu-id="9aae6-117">Способ обработки исключений, реализуемый в .NET, обладает следующими преимуществами:</span><span class="sxs-lookup"><span data-stu-id="9aae6-117">The way .NET implements exception handling provides the following advantages:</span></span>

- <span data-ttu-id="9aae6-118">Создание и обработка исключений работают одинаково для языков программирования .NET.</span><span class="sxs-lookup"><span data-stu-id="9aae6-118">Exception throwing and handling works the same for .NET programming languages.</span></span>

- <span data-ttu-id="9aae6-119">Не требует определенного синтаксиса языка для обработки исключений, а позволяет каждому языку определить собственный синтаксис.</span><span class="sxs-lookup"><span data-stu-id="9aae6-119">Doesn't require any particular language syntax for handling exceptions, but allows each language to define its own syntax.</span></span>

- <span data-ttu-id="9aae6-120">Исключения можно создавать между разными процессами и даже компьютерами.</span><span class="sxs-lookup"><span data-stu-id="9aae6-120">Exceptions can be thrown across process and even machine boundaries.</span></span>

- <span data-ttu-id="9aae6-121">В приложение можно добавить код обработки исключений для повышения надежности программы.</span><span class="sxs-lookup"><span data-stu-id="9aae6-121">Exception-handling code can be added to an application to increase program reliability.</span></span>

<span data-ttu-id="9aae6-122">Исключения обеспечивают ряд преимуществ по сравнению с другими методами уведомления об ошибках, например кодами возврата.</span><span class="sxs-lookup"><span data-stu-id="9aae6-122">Exceptions offer advantages over other methods of error notification, such as return codes.</span></span> <span data-ttu-id="9aae6-123">Сбои не остаются незамеченными, так как среда выполнения завершает работу приложения при наличии необработанного исключения.</span><span class="sxs-lookup"><span data-stu-id="9aae6-123">Failures don't go unnoticed because if an exception is thrown and you don't handle it, the runtime terminates your application.</span></span> <span data-ttu-id="9aae6-124">Недопустимые значения не распространяются по системе из-за того, что код не способен выполнить проверку кода возврата ошибки.</span><span class="sxs-lookup"><span data-stu-id="9aae6-124">Invalid values don't continue to propagate through the system as a result of code that fails to check for a failure return code.</span></span>
